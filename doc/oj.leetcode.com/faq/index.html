Why are my solutions missing?

The code you typed into the code editor is stored temporarily in your browser session, however the code you submitted will be stored permanently.
In the submissions page, click on the status of that submission row to view more details such as your submitted code.

Am I allowed to print something to stdout?

In most of the cases, no. Do not print anything to stdout in your code, or else you will most likely get Wrong Answer, Output Limit Exceeded or Runtime Error. If you want to debug something quickly and you know what you are doing, then by all means go ahead.

What is the difference between 'Time Limit Exceeded' and 'Timeout'?

If your solution is judged 'Time Limit Exceeded', it could be:

  1. Your code has an underlying infinite loop.
  2. Your algorithm is too slow and has a high time complexity.

On the other hand, 'Timeout' simply means the server queue is busy and could not process your submission for the time being. Please wait for about 10 seconds and submit again.

Why is my output wrong answer even though it looks identical to expected?

Most probably the string you returned from your code has some invisible ASCII characters. These ASCII characters (such as NUL) are not printable, and therefore you will not see them in the output.

Why does my code produce a different output compared to my local environment?

First of all, LeetCode Judger executes all test cases using the same program instance, which means you need to check the following:

  1. Are there any global or static variables declared? Please avoid these variables if possible. If you must declare one, remember to reset them either in the default constructor or in the first line of your called method. See this Discuss thread for more details.
  2. Most likely there is a bug in your code. The code could have worked perfectly fine in your local environment due to pure luck. See this Discuss thread for an example.
  3. In some rare cases when your code encounters a Runtime Error, LeetCode OJ might provide you with an incorrect failed test case. If this happens to you, please let us know.

I messed up in the code editor and would like to start from scratch on a particular problem. How do I reload the default code definition?

Delete every single character in the code editor, then reload the page.

I have already signed up at leetcode.com, why can't I sign in to OJ?

You would need to create a new account at oj.leetcode.com. There is no plan to support account integration across these two sites.
